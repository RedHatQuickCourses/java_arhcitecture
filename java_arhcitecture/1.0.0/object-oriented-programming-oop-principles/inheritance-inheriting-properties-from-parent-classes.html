Inheritance

Inheritance

Inheritance is a fundamental Object-Oriented Programming (OOP) principle in Java that enables classes to acquire properties and behaviors from other classes, promoting code reuse and establishing hierarchical relationships. This section details how inheritance works, its benefits, and provides a hands-on activity to solidify your understanding.

Understanding Inheritance in Java

Inheritance is a mechanism where one class acquires the properties (fields) and behaviors (methods) of another class. This allows for code reusability and the creation of a natural "is-a" relationship between different types of objects. For instance, a Car is a Vehicle, and a Motorcycle is a Vehicle.

In Java, the class that is being inherited from is called the superclass, parent class, or base class. The class that inherits the properties and behaviors is called the subclass, child class, or derived class.

The core idea is to extend the functionality of an existing class without having to rewrite its code. A child class can use the methods and fields defined in its parent, and it can also:

  • Add new fields and methods specific to its own type.

  • Override inherited methods to provide a specialized implementation for its own context.

Benefits of Inheritance

Inheritance provides several key advantages in software design and development:

  • Code Reusability: The most significant benefit is the ability to reuse code. Common logic, fields, and methods can be defined once in a superclass and then inherited by multiple subclasses, reducing redundancy and making the codebase easier to maintain.

  • Polymorphism: Inheritance is a prerequisite for achieving runtime polymorphism through method overriding. This allows objects of different subclasses to be treated as objects of their common superclass, enabling flexible and extensible designs.

  • Hierarchical Classification: It helps to establish a clear and logical hierarchy among classes, mirroring real-world categorizations. This makes the system more organized and understandable.

  • Extensibility: New features or variations can be introduced by creating new subclasses that extend existing ones. This allows for adding functionality without modifying the core logic of the parent class, adhering to the Open/Closed Principle (open for extension, closed for modification).

Types of Inheritance in Java

Java supports specific types of inheritance between classes, and a form of multiple inheritance through interfaces:

  • Single Inheritance: A class inherits from only one parent class. This is the most straightforward and common form of inheritance.

    Parent Class → Child Class

  • Multilevel Inheritance: A subclass inherits from a parent class, which itself is a subclass of another class. This creates a chain of inheritance.

    Grandparent Class → Parent Class → Child Class

  • Hierarchical Inheritance: Multiple subclasses inherit from a single parent class. This models a situation where several specialized classes stem from a common generalized class.

    Parent Class

    / \

    Child Class 1 Child Class 2

  • Multiple Inheritance (via Interfaces): Java does not support multiple inheritance of classes (a class cannot extends more than one class) to avoid complex issues like the "diamond problem." However, it allows a class to implement multiple interfaces. While interfaces only define abstract methods (or default/static methods since Java 8), this mechanism provides a way for a class to inherit multiple type behaviors.

Technical Explanation: How Inheritance Works

In Java, the extends keyword is used to declare that a class is a subclass of another class. When a class B extends class A, class B inherits all non-private fields and methods from class A. Private members of the superclass are not directly accessible by the subclass but can be accessed indirectly through public or protected methods provided by the superclass.

Consider the following example demonstrating a Vehicle superclass and a Car subclass:

// Vehicle.java (Superclass)
class Vehicle {
    protected String brand = "Generic"; // Using 'protected' for direct subclass access
    int numberOfWheels; // Default (package-private) access

    public Vehicle(int wheels) {
        this.numberOfWheels = wheels;
    }

    public void honk() {
        System.out.println("Tuut, tuut!");
    }

    public void drive() {
        System.out.println("A " + brand + " vehicle with " + numberOfWheels + " wheels is moving.");
    }

    // A method to get the brand, accessible by subclasses and outside
    public String getBrand() {
        return brand;
    }
}

// Car.java (Subclass)
class Car extends Vehicle { // Car inherits from Vehicle
    String modelName = "Sedan";
    String fuelType;

    public Car(String brand, String modelName, String fuelType) {
        super(4); // Call the superclass constructor, a car typically has 4 wheels
        this.brand = brand; // Inherited protected field can be directly accessed
        this.modelName = modelName;
        this.fuelType = fuelType;
    }

    public void displayCarInfo() {
        System.out.println("Brand: " + getBrand()); // Accessing inherited method
        System.out.println("Model: " + modelName);
        System.out.println("Fuel Type: " + fuelType);
        honk(); // Calling inherited method
    }

    // Method Overriding: Providing a specialized implementation for an inherited method
    @Override // Annotation to indicate method overriding (good practice)
    public void drive() {
        System.out.println("The " + brand + " " + modelName + " car, running on " + fuelType + ", is driving smoothly on the road.");
    }

    public void activateWipers() {
        System.out.println("Wipers activated for the " + modelName + ".");
    }
}

// InheritanceDemo.java (Main class to demonstrate)
public class InheritanceDemo {
    public static void main(String[] args) {
        System.out.println("--- Demonstrating Inheritance ---");

        // Create a Vehicle object
        Vehicle genericVehicle = new Vehicle(6); // A truck perhaps
        System.out.println("\n== Generic Vehicle ==");
        System.out.println("Vehicle brand: " + genericVehicle.getBrand());
        genericVehicle.honk();
        genericVehicle.drive();

        // Create a Car object
        Car myCar = new Car("Tesla", "Model 3", "Electric");
        System.out.println("\n== My Car ==");
        myCar.displayCarInfo(); // Calls Car's own method, which uses inherited getBrand() and honk()
        myCar.drive();          // Calls Car's overridden drive() method
        myCar.activateWipers(); // Calls Car's own specific method

        // Demonstrating an inherited method by treating Car as Vehicle (Polymorphism)
        System.out.println("\n== Car as a Vehicle ==");
        Vehicle anotherVehicle = new Car("Ford", "F-150", "Gasoline");
        anotherVehicle.drive(); // Still calls the Car's overridden drive() method!
        // anotherVehicle.activateWipers(); // This would cause a compile-time error!
                                          // Vehicle type does not have activateWipers()
    }
}

In this example:

  • The Car class extends the Vehicle class, automatically inheriting brand, numberOfWheels, honk(), drive(), and getBrand().

  • The Car constructor uses super(4) to call the Vehicle class constructor, initializing numberOfWheels.

  • The Car class can directly access the protected brand field and sets its value.

  • myCar.displayCarInfo() calls getBrand() and honk(), demonstrating the reuse of inherited methods.

  • The Car class introduces its own specific fields (modelName, fuelType) and methods (activateWipers()).

  • The drive() method is overridden in the Car class. When myCar.drive() is called, the specialized Car version is executed, not the Vehicle’s generic version. The `@Override annotation is a compiler hint and good practice.

  • The last part of main demonstrates that a Car object can be assigned to a Vehicle reference (Vehicle anotherVehicle = new Car(…​)). When anotherVehicle.drive() is called, the runtime still invokes the Car’s overridden `drive() method due to polymorphism. However, activateWipers() cannot be called using the Vehicle reference because it’s a Car-specific method not present in the Vehicle type.

Hands-on Activity: Exploring Vehicle Hierarchy with Inheritance

This activity will guide you through creating and compiling Java code that defines a simple hierarchy of vehicles using inheritance. You will observe how subclasses inherit common behaviors and can implement specialized functionalities.

  1. Set up your development environment: Ensure you have a Java Development Kit (JDK) installed and configured, and a text editor or IDE.

  2. Create the Vehicle.java file: Create a file named Vehicle.java in a new directory (e.g., inheritance_lab) and add the following code for the superclass: [source,java] ---- // Vehicle.java (Superclass) class Vehicle { protected String type; protected int maxSpeed;

    public Vehicle(String type, int maxSpeed) {
    this.type = type;
    this.maxSpeed = maxSpeed;
    System.out.println("Initializing a " + type + ".");
}
    public void start() {
    System.out.println(type + " is starting its engine.");
}
    public void stop() {
    System.out.println(type + " is stopping its engine.");
}
    public void accelerate() {
    System.out.println(type + " is accelerating up to " + maxSpeed + " km/h.");
}
        public void displayInfo() {
        System.out.println("Type: " + type + ", Max Speed: " + maxSpeed + " km/h.");
    }
}
----

  3. Create the Car.java file: Create a file named Car.java in the same directory. This class will extend Vehicle and provide specialized methods and an overridden accelerate() method. [source,java] ---- // Car.java (Subclass) class Car extends Vehicle { private int numberOfDoors;

    public Car(String brand, int maxSpeed, int numberOfDoors) {
    super(brand + " Car", maxSpeed); // Call Vehicle constructor
    this.numberOfDoors = numberOfDoors;
}
    @Override
public void accelerate() {
    System.out.println(type + " smoothly accelerates to " + maxSpeed + " km/h. Enjoy the ride!");
}
    public void openTrunk() {
    System.out.println(type + " trunk is opening.");
}
        // Overload displayInfo for Car-specific details
    public void displayCarInfo() {
        super.displayInfo(); // Call parent's displayInfo
        System.out.println("Number of doors: " + numberOfDoors);
    }
}
----

  4. Create the Motorcycle.java file: Create another subclass file named Motorcycle.java in the same directory. It also extends Vehicle and overrides accelerate(). [source,java] ---- // Motorcycle.java (Subclass) class Motorcycle extends Vehicle { private boolean hasSidecar;

    public Motorcycle(String brand, int maxSpeed, boolean hasSidecar) {
    super(brand + " Motorcycle", maxSpeed); // Call Vehicle constructor
    this.hasSidecar = hasSidecar;
}
    @Override
public void accelerate() {
    System.out.println(type + " roars and accelerates to " + maxSpeed + " km/h! Feel the wind!");
}
    public void performWheelie() {
    if (!hasSidecar) {
        System.out.println(type + " is performing a wheelie!");
    } else {
        System.out.println(type + " is too stable for a wheelie.");
    }
}
        // Overload displayInfo for Motorcycle-specific details
    public void displayMotorcycleInfo() {
        super.displayInfo(); // Call parent's displayInfo
        System.out.println("Has sidecar: " + hasSidecar);
    }
}
----

  5. Create the InheritanceLab.java file: Finally, create a main class InheritanceLab.java in the same directory to orchestrate the demonstration of these vehicle objects. [source,java] ---- // InheritanceLab.java (Main class for demonstration) public class InheritanceLab { public static void main(String[] args) { System.out.println("=== Java Inheritance Lab ===");

    // 1. Demonstrate a generic Vehicle
System.out.println("\n--- Generic Vehicle Actions ---");
Vehicle generalVehicle = new Vehicle("Standard Transport", 100);
generalVehicle.displayInfo();
generalVehicle.start();
generalVehicle.accelerate();
generalVehicle.stop();
    // 2. Demonstrate a Car
System.out.println("\n--- Car Actions ---");
Car myCar = new Car("Toyota", 200, 4);
myCar.displayCarInfo(); // Car's specific display method
myCar.start();          // Inherited method
myCar.accelerate();     // Overridden method
myCar.openTrunk();      // Car's specific method
myCar.stop();           // Inherited method
    // 3. Demonstrate a Motorcycle
System.out.println("\n--- Motorcycle Actions ---");
Motorcycle myMotorcycle = new Motorcycle("Harley", 180, false);
myMotorcycle.displayMotorcycleInfo(); // Motorcycle's specific display method
myMotorcycle.start();                 // Inherited method
myMotorcycle.accelerate();            // Overridden method
myMotorcycle.performWheelie();        // Motorcycle's specific method
myMotorcycle.stop();                  // Inherited method
            Motorcycle sidecarBike = new Motorcycle("BMW", 150, true);
        System.out.println("\n--- Motorcycle with Sidecar Actions ---");
        sidecarBike.displayMotorcycleInfo();
        sidecarBike.performWheelie(); // See different behavior due to condition
    }
}
----

  6. Compile the Java files: Open your terminal or command prompt, navigate to the inheritance_lab directory, and compile all the Java files. [source,bash] ---- javac Vehicle.java Car.java Motorcycle.java InheritanceLab.java ---- If successful, you will see no output, and .class files will be generated for each .java file.

  7. Execute the InheritanceLab class: Run the main class to observe the inheritance in action. [source,bash] ---- java InheritanceLab ----

This activity vividly illustrates how:

  • Car and Motorcycle objects reuse the start() and stop() methods from the Vehicle superclass.

  • The accelerate() method demonstrates polymorphism, executing different code based on the actual object type (Vehicle, Car, or Motorcycle).

  • Subclasses can introduce their own unique features (openTrunk for Car, performWheelie for Motorcycle).

  • Constructors in subclasses utilize super() to initialize the superclass portion of the object.
