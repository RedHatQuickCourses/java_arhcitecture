Object-Oriented Programming (OOP) Principles

Polymorphism: Using One Name for Multiple Actions

In the realm of Object-Oriented Programming (OOP), polymorphism is a fundamental principle that allows objects to take on many forms. The term "polymorphism" is derived from Greek words: "poly" meaning many, and "morph" meaning forms. In Java, this translates to the ability to use "one name for multiple actions," as highlighted in our objectives.

Polymorphism enables a single interface to represent different underlying forms or data types. This significantly enhances the flexibility, extensibility, and maintainability of your code by promoting generic programming and reducing code coupling.

Java primarily supports two types of polymorphism:

  1. Compile-time Polymorphism (Method Overloading)

  2. Runtime Polymorphism (Method Overriding)

Compile-time Polymorphism: Method Overloading

Compile-time polymorphism, also known as static polymorphism, is achieved through method overloading. Method overloading allows a class to have multiple methods with the same name, provided they have different method signatures. A method signature is defined by the method’s name and the number, type, and order of its parameters. The return type of the method is not part of the method signature and therefore cannot be used to differentiate overloaded methods.

During compilation, the Java compiler determines which overloaded method to invoke based on the arguments provided in the method call. This decision is made statically at compile time, hence the name "compile-time polymorphism."

Key characteristics of Method Overloading:

  • Same Method Name: All overloaded methods share the same name within the same class.

  • Different Method Signatures: The parameter list must differ in at least one of these ways:

  • Number of parameters.

  • Data type of parameters.

  • Order of parameters (if types are different).

  • Return Type: The return type can be different but is not sufficient to distinguish overloaded methods. If only the return type differs, it will result in a compile-time error.

Example of Method Overloading

Consider a Calculator class that needs to perform addition on various data types or with a different number of operands.

class Calculator {
    // Method to add two integers
    public int add(int a, int b) {
        return a + b;
    }

    // Overloaded method to add three integers
    public int add(int a, int b, int c) {
        return a + b + c;
    }

    // Overloaded method to add two doubles
    public double add(double a, double b) {
        return a + b;
    }

    // Overloaded method to concatenate two strings (conceptual addition)
    public String add(String s1, String s2) {
        return s1 + s2;
    }
}

public class OverloadingDemo {
    public static void main(String[] args) {
        Calculator calc = new Calculator();

        System.out.println("Adding two integers: " + calc.add(10, 20));
        System.out.println("Adding three integers: " + calc.add(10, 20, 30));
        System.out.println("Adding two doubles: " + calc.add(10.5, 20.3));
        System.out.println("Concatenating strings: " + calc.add("Hello", " World"));
    }
}

In the example above, the add method is overloaded four times. The compiler correctly identifies and calls the appropriate add method based on the arguments passed during the main method execution.

Runtime Polymorphism: Method Overriding

Runtime polymorphism, also known as dynamic polymorphism, is achieved through method overriding. Method overriding occurs when a subclass (child class) provides a specific implementation for a method that is already defined in its superclass (parent class). When a method is overridden, the method call is resolved at runtime based on the actual object type, not the reference type.

Key characteristics of Method Overriding:

  • Inheritance: Method overriding can only occur in an inheritance hierarchy, where a subclass extends a superclass.

  • Same Method Signature: The overriding method in the subclass must have the exact same method name, return type, and parameter list as the method in the superclass.

  • @Override Annotation: It is good practice to use the @Override annotation in the subclass. This annotation helps the compiler verify that the method is indeed an override, preventing common errors like typos in method names or incorrect parameter lists.

  • Access Modifier: The access modifier of the overriding method must be the same or have broader access than the overridden method in the superclass (e.g., protected to public is allowed, but public to private is not).

The power of runtime polymorphism lies in upcasting. When a superclass reference variable refers to an object of a subclass, calling an overridden method through that reference will execute the subclass’s version of the method. This allows you to write generic code that can work with objects of different classes that share a common superclass, without needing to know their specific type at compile time.

Example of Method Overriding

Consider a Shape class with a draw() method. Different shapes (like Circle and Rectangle) will have their own way of drawing themselves.

class Shape {
    public void draw() {
        System.out.println("Drawing a generic shape.");
    }
}

class Circle extends Shape {
    @Override // Good practice to include this annotation
    public void draw() {
        System.out.println("Drawing a Circle.");
    }
}

class Rectangle extends Shape {
    @Override
    public void draw() {
        System.out.println("Drawing a Rectangle.");
    }
}

public class OverridingDemo {
    public static void main(String[] args) {
        Shape s1 = new Circle(); // Upcasting: Shape reference to Circle object
        Shape s2 = new Rectangle(); // Upcasting: Shape reference to Rectangle object
        Shape s3 = new Shape(); // Shape reference to Shape object

        s1.draw(); // Calls Circle's draw() method
        s2.draw(); // Calls Rectangle's draw() method
        s3.draw(); // Calls Shape's draw() method

        // Demonstrating an array of shapes
        Shape[] shapes = new Shape[3];
        shapes[0] = new Circle();
        shapes[1] = new Rectangle();
        shapes[2] = new Shape();

        System.out.println("\nDrawing shapes from an array:");
        for (Shape shape : shapes) {
            shape.draw(); // Each call executes the appropriate draw() method
        }
    }
}

In this example, even though s1 and s2 are declared as Shape type references, the actual methods invoked for s1.draw() and s2.draw() are determined at runtime based on the actual object they refer to (Circle and Rectangle respectively). This is runtime polymorphism in action.

Benefits of Polymorphism

Polymorphism is a powerful OOP concept that offers several advantages:

  • Flexibility and Extensibility: New classes can be added without modifying existing code, as long as they adhere to the common interface or extend the same base class. This makes applications easier to extend and adapt to changing requirements.

  • Code Reusability: Generic code can be written to work with objects of different classes, promoting reusability and reducing redundant code.

  • Improved Readability and Maintainability: By abstracting away specific implementations, code becomes cleaner and easier to understand. Changes to specific implementations only affect that subclass, not the generic code that uses it.

  • Decoupling: It helps decouple code, meaning components are less dependent on each other, which simplifies testing and development.

Hands-on Lab: Exploring Polymorphism

In this lab, you will implement and observe both method overloading and method overriding in Java.

Activity 1: Method Overloading with a Greeter Class

In this activity, you’ll create a simple Greeter class that uses method overloading to greet people in different ways.

Objectives

  • Create a class with overloaded methods.

  • Understand how the compiler selects the correct method based on arguments.

Instructions

  1. Create a new directory for your lab: bash mkdir polymorphism-lab cd polymorphism-lab

  2. Create a file named Greeter.java and populate it with the following code: [source,java] ---- // Greeter.java class Greeter { // Greet with a default message public void sayHello() { System.out.println("Hello there!"); }

    // Greet a specific person by name
public void sayHello(String name) {
    System.out.println("Hello, " + name + "!");
}
    // Greet a specific person a given number of times
public void sayHello(String name, int times) {
    for (int i = 0; i < times; i++) {
        System.out.println("Hello, " + name + "!");
    }
}
        // Greet multiple people (an array of names)
    public void sayHello(String[] names) {
        System.out.println("--- Greeting a group ---");
        for (String name : names) {
            System.out.println("Hello, " + name + "!");
        }
        System.out.println("------------------------");
    }
}
    // Main class to test the Greeter
public class OverloadingLab {
    public static void main(String[] args) {
        Greeter myGreeter = new Greeter();
    System.out.println("--- Demonstrating Method Overloading ---");
    myGreeter.sayHello(); // Calls sayHello() with no arguments
myGreeter.sayHello("Alice"); // Calls sayHello(String name)
myGreeter.sayHello("Bob", 3); // Calls sayHello(String name, int times)
    String[] teamMembers = {"Charlie", "David", "Eve"};
myGreeter.sayHello(teamMembers); // Calls sayHello(String[] names)
            System.out.println("\nOverloading demonstration complete.");
    }
}
----

  3. Compile your Java code: bash javac Greeter.java OverloadingLab.java

  4. Execute the compiled code: bash java OverloadingLab

Hello, Bob!

Review

Observe how the sayHello method is invoked with different arguments, and the Java compiler correctly selects the appropriate version of the method based on the parameter list. This showcases compile-time polymorphism.

In this activity, you will implement method overriding to demonstrate runtime polymorphism using a Vehicle hierarchy.

Objectives

  • Observe runtime polymorphism through upcasting.

  2. Create a file named Vehicle.java and add the superclass definition: [source,java] ---- // Vehicle.java class Vehicle { public void startEngine() { System.out.println("Vehicle engine starts with a generic sound."); }
