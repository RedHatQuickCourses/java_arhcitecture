In this session, we are going to explore the core principles of Object-Oriented Programming, commonly known as OOP. At its heart, this programming paradigm is built around the concept of "objects," which are structures that can contain both data and the code to manipulate that data.

There are four fundamental principles of OOP that you should be familiar with. First, there is abstraction, which is the practice of focusing on the essential elements of an object while intentionally ignoring unnecessary details. Next, we have encapsulation, where we bundle data and the methods that operate on that data into a single unit. Then, there is inheritance, a mechanism that allows a class to derive or inherit properties from a parent class. Finally, polymorphism allows us to use a single name or interface to represent multiple different actions.

Now, let's look at how these objects are actually structured. Every object is composed of two primary components. First, there is the data, which is stored in what we call instance fields. Second, there are responsibilities, which are defined by methods that perform specific actions on the object's internal data.

Before we conclude, it is helpful to understand how Java fits into this paradigm. You will often hear Java described as being "partially object-oriented." The reason for this is that Java supports primitive data types—such as `int` or `float`—alongside its objects. In a purely object-oriented language, every single element would be an object, and there would be no support for these primitive types. Understanding this distinction is key to mastering how Java handles memory and data structures.
