    • Object-Oriented Programming (OOP) Principles
      • Abstraction: Collecting required elements while ignoring unnecessary details.
      • Encapsulation: Bundling data and methods inside an object.
      • Inheritance: Inheriting properties from parent classes.
      • Polymorphism: Using one name for multiple actions.
      • Object Structure: Objects consist of data (instance fields) and responsibilities (methods).
      • Java’s Nature: Considered partially object-oriented because it supports primitive data types like int and float.
    • Java Platform Components: JDK, JRE, and JVM
      • Java Development Kit (JDK): The outermost layer and a superset containing all tools necessary for development.
      • Java Runtime Environment (JRE): Located inside the JDK, providing libraries (like java.lang and java.io) to run programs.
      • Java Virtual Machine (JVM): The innermost component and actual executor; it is platform-dependent for different operating systems.
      • Compilation Flow: Java source code is compiled into bytecode (.class files) by the JDK and then executed by the JVM.
    • Java Development and Monitoring Tools
      • Compiler (javac): Translates Java code into class files.
      • Application Launcher (java): Used to execute the compiled class files.
      • Decompiler (javap): Used to view the details of a .class file.
      • Archiver (jar): Bundles multiple files into a single archive or unzips existing ones.
      • Debugger (jdb): A utility for debugging Java code.
      • Security Tool (keytool): Creates key stores and trust stores for SSL/HTTPS security.
      • Monitoring Tools: JConsole and JVisualVM are used to monitor memory, threads, and process performance.
      • Java Flight Recorder (JFR): A tool for troubleshooting complex issues by storing data over intervals.

    • Enterprise Java and Specifications
      • Definition of Specification: Guidelines or rules that define what should be done without prescribing the exact implementation.
      • Implementation Examples: Red Hat implements specifications through JBoss EAP, while Oracle uses WebLogic.
      • JBoss EAP Versioning: EAP 8 supports JDK 11, 17, and 21, but does not support JDK 8 (which was supported by EAP 7.4).
