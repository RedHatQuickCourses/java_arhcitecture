#  Application Launcher (java)

[[java-application-launcher]]
= Java Application Launcher (`java`)

In the Java development workflow, once your human-readable Java source code (`.java` files) has been successfully transformed into platform-independent bytecode (`.class` files) by the `javac` compiler, the next crucial step is to run your application. This is where the `java` command, often referred to as the **Application Launcher**, comes into play.

The `java` command is the primary utility for executing compiled Java bytecode. It acts as the bridge between your compiled `.class` files and the Java Virtual Machine (JVM), which is responsible for interpreting and executing the bytecode on the underlying operating system. As the context states, "Java is the, what is Java? This is the application launcher, you have that .classfile, you want to execute that, you will execute it with the .java." (lines 146-149).

When you invoke the `java` command with the name of your compiled class, it performs several key functions:

. **Locates the `.class` file**: The `java` command searches for the specified `.class` file within the current directory or the defined classpath.
. **Loads the class**: Once located, the class loader (a component of the Java Runtime Environment or JRE) loads the bytecode into memory. The JRE provides the necessary libraries to run programs.
. **Verifies the bytecode**: The bytecode verifier, also part of the JRE, checks the bytecode for format correctness and security vulnerabilities before execution.
. **Initializes the JVM**: If not already running, the `java` command initiates a new instance of the Java Virtual Machine. As we learned, the JVM is the innermost component and the actual executor, and it is platform-dependent (lines 114-115).
. **Passes to JVM**: The loaded and verified bytecode is then passed to the JVM for execution. The JVM's Just-In-Time (JIT) compiler may further optimize the bytecode into native machine code for faster execution.
. **Executes the `main` method**: The JVM specifically looks for and executes the `public static void main(String[] args)` method within the specified class, which serves as the entry point for standalone Java applications.

This process directly supports the overall compilation flow: "Java source code is compiled into bytecode (.class files) by the JDK and then executed by the JVM." The `java` command is the direct mechanism to initiate this execution phase by the JVM.

== Hands-on Lab: Executing Compiled Java Code

This lab will guide you through the process of writing a simple Java program, compiling it, and then executing it using the `java` application launcher.

=== Objectives

*   Write a basic "Hello, World!" Java program.
*   Compile the Java source code using `javac`.
*   Execute the compiled bytecode using the `java` command.

=== Prerequisites

*   A Java Development Kit (JDK) installed on your system.
*   A text editor (e.g., VS Code, Sublime Text, Notepad++, or even a simple `nano`/`vim` on Linux).
*   A command-line terminal.

=== Steps

. **Create a Java Source File**
    . Open your preferred text editor.
    . Type the following Java code:
    +
    ```java
    // Greeting.java
    public class Greeting {
        public static void main(String[] args) {
            System.out.println("Hello from the Java Application Launcher!");
        }
    }
    ```
    . Save the file as `Greeting.java` in a directory of your choice (e.g., `~/javaprojects/` on Linux/macOS or `C:\javaprojects\` on Windows).
    +
    [NOTE]
    The filename `Greeting.java` must exactly match the public class name `Greeting` for Java to compile correctly.

. **Compile the Java Source Code**
    . Open your command-line terminal.
    . Navigate to the directory where you saved `Greeting.java`.
    +
    For example:
    ```bash
    cd ~/javaprojects/
    ```
    . Compile the `Greeting.java` file using the `javac` command (the compiler tool provided by the JDK):
    ```bash
    javac Greeting.java
    ```
    . If the compilation is successful, a new file named `Greeting.class` will be created in the same directory. This `Greeting.class` file contains the platform-independent bytecode generated by the compiler.
    +
    [TIP]
    If you encounter errors, carefully double-check your code for typos or syntax mistakes. Common errors include missing semicolons, mismatched curly braces, or incorrect class names.

. **Execute the Compiled Java Code**
    . While still in the same directory in your terminal, use the `java` command to execute the `Greeting.class` file.
    . *Important*: When using the `java` command, you specify the *class name* (e.g., `Greeting`), not the `.class` file extension. The `java` command will automatically find and load the corresponding `.class` file.
    ```bash
    java Greeting
    ```
    . You should see the following output in your terminal:
    ```
    Hello from the Java Application Launcher!
    ```

Congratulations! You have successfully used the `java` application launcher to run your first compiled Java program. This fundamental workflow of compiling with `javac` and executing with `java` is at the core of developing and running standalone Java applications.

[discrete]
=== Further Exploration

*   Try modifying the message in `Greeting.java`, recompile with `javac`, and then re-execute with `java` to observe the changes.
*   Experiment with passing command-line arguments to your `main` method. For instance, if you changed `System.out.println("Hello from the Java Application Launcher!");` to `System.out.println("Hello, " + args[0] + "!");`, you could then run `java Greeting World` and see "Hello, World!" as output.
